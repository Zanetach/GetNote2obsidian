# GetNote Sync for Obsidian

把 Get笔记里的 AI 转写内容同步到 Obsidian。

## 当前能力

- 通过 Get笔记 OpenAPI 拉取新笔记
- 将笔记写入指定的 Obsidian 文件夹
- 可按日期自动分目录
- 可按 `source` / `note_type` 过滤，更适合录音卡场景
- 支持把发芽内容同步到与核心笔记相同的目录，并在标题前加 `[发芽]`
- 支持核心笔记与发芽报告自动互链
- 支持自动生成 `主题/xxx.md` 索引页，按主题聚合内容
- 支持在核心录音笔记里保留原始转写，并可折叠显示
- 核心录音笔记默认按“会议纪要”结构展示：会议摘要、讨论主题、行动项、讨论过程、原始转写
- 如果核心录音还没有对应发芽笔记，插件会给出“发芽提示”
- 如果你在 Obsidian 里手动删除了已同步笔记，插件会记住这次删除，后续不再自动恢复
- 支持手动同步
- 支持刷新已导入笔记，按最新模板重写旧笔记
- 支持启动自动同步
- 支持定时轮询
- 保存 `since_id`，避免重复导入
- 支持基于本地设备机器码的 License 授权，兼容桌面端和移动端

## 使用前准备

1. 到 `https://www.biji.com/openapi` 创建应用
2. 获取 `Client ID` 和 `API Key`
3. 在 Obsidian 的社区插件开发目录中放入本项目

## 开发

```bash
npm install
npm run build
```

把以下文件复制到你的 vault 插件目录：

- `manifest.json`
- `main.js`
- `versions.json`

例如：

```text
<你的 Vault>/.obsidian/plugins/getnote-sync/
```

## License 授权

- 插件会在每台设备本地生成一个 `deviceId`，并保存在 Obsidian 的 `SecretStorage`
- 这个 `deviceId` 不写入 vault 同步文件，更适合移动端和多设备场景
- 设置页会显示当前设备机器码和授权状态
- 插件内只保留公钥；真正签发客户授权码使用你的内部私有发码工具
- 授权码格式为 `gns1.<payload>.<signature>`
- 一条授权码可以绑定多个 `deviceId`，适合同时覆盖 Mac / Windows / 手机 / 平板
- 默认最多绑定 3 台设备，但生成时可以显式选择“无限设备”
- 旧的本机测试授权码仍然兼容，但不建议继续用于客户分发

## 说明

Get笔记公开文档里明确给出了笔记列表接口示例，但公开页面没有完全展开所有返回字段示例，所以当前插件对返回 JSON 做了宽容解析：

- 正文候选字段默认尝试 `markdown, content, body, text, transcript, transcription, note_content`
- 摘要候选字段默认尝试 `summary, abstract, ai_summary, digest`

如果你的真实接口字段名不同，可以直接在插件设置里调整。

## 录音卡建议配置

- 开启 `只同步匹配筛选规则的笔记`
- 开启 `同步发芽报告`
- `允许的 Source 关键词` 先填：`record,audio,recorder,getseed,device,bluetooth`
- `允许的 Note Type 关键词` 先填：`audio,voice,transcript,record`
- `主题索引文件夹` 保持默认 `主题`

如果后面你从 Get笔记里拿到一条真实录音卡笔记的 JSON，还可以继续把过滤规则调得更准。

## 推荐组织方式

- `GetNote Inbox/...`：核心录音转写笔记与发芽笔记
- `主题/...`：主题索引页，分别聚合核心笔记和发芽报告

这样更适合把录音笔记当作“事实源”，把发芽报告当作“扩散层”，再通过主题页回收整理。

## 手动删除后的处理

- 已同步到 Obsidian 的笔记如果被你手动删除，插件会把它视为“用户主动不要了”
- 后续自动同步或重新拉取时，不会把这条笔记偷偷导回来
- 如果你想恢复这些笔记，可以先点一次 `清空已删除记录`，再执行 `重置同步游标`
